## 核心心法：逆向思维 (Reverse Engineering)

当你面对一个完全陌生的功能（例如：我要改移速，但我不知道改哪里），请严格遵循以下三步法：

### Step 1: 猜开发者的思路 (Guess)
**提问**: "如果我是 UE4 开发者，我用什么组件控制这个功能？"
- 不要想 "内存偏移是多少"，要想 "类名是什么"。
- **移速** -> `CharacterMovement`? `Velocity`?
- **血量** -> `HealthComponent`? `AttributeSet`? `Pawn.Health`?
- **子弹** -> `WeaponComponent`? `Ammo`?

### Step 2: 查正经文档 (Research)
**动作**: 不要去搜外挂论坛，去搜 **UE4 官方文档** 或 **开发者教程**。
- **Google/ChatGPT**: "UE4 C++ change character walk speed"
- **Answer**: 几乎都在 `UCharacterMovementComponent::MaxWalkSpeed`。

### Step 3: 找对应 Offset (Locate)
**工具**: `SDK.txt` (UE4Dumper Dump 出的 SDK)
1. 打开 `docs/SDK.txt`
2. 搜索 **类名** (e.g., `CharacterMovementComponent`) -> 确认存在。
3. 搜索 **变量名** (e.g., `MaxWalkSpeed`) -> 拿到 Offset (e.g., `0x18C`)。
4. 验证层级：`ACharacter` -> `0x288` (MovementComp) -> `0x18C` (Speed)。

## 实战案例：移速 (Speed Hack)
1. **Guess**: 名字里肯定带 Speed。
2. **Research**: 查到 `MaxWalkSpeed`。
3. **Locate**:
   - `SDK.txt` 搜 `0x288` -> 确认为 `CharacterMovement` 指针。
   - `SDK.txt` 搜 `MaxWalkSpeed` -> 确认为 `0x18c`。
