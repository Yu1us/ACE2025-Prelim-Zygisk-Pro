## 严重问题

### 1. `registry.hpp` - 内存所有权模型有隐患
`createAll()` 返回结果误导。

```cpp
// L22-28
std::vector<IHackModule *> createAll() {
    std::vector<IHackModule *> modules;
    for (auto &f : factories_) {
        modules.push_back(f());  // f() 返回 static 局部变量的指针
    }
    return modules;
}
```
**分析**: `AutoRegister` lambda 使用 `static T instance`，返回静态局部变量指针。不会内存泄漏，但 `createAll()` 暗示每次创建新实例。多次调用得到同一套指针，可能状态混乱。

**建议**: 实际是单例模式，改名 `getAll()` 或 `modules()`。

### 3. `mod_aim.cpp` - 命名空间与全局规范不统一
Aimbot 模块偏移量管理应与其他模块对齐。其他模块（speed, gunoffset, projectile）用 `Game::` 命名空间（来自 `game/offsets.hpp`）。
```cpp
namespace Offsets { constexpr uintptr_t AimbotPatch = 0x670F3F8; }
namespace ARM64 { constexpr uint8_t NOP[] = {0x1F, 0x20, 0x03, 0xD5}; }
```
**建议**:
- `AimbotPatch` 移入 `game/offsets.hpp`
- `ARM64::NOP` 移入公共头文件 `utils.hpp`

### 4. `mod_aim.cpp` - `set_memory_protection()` 工具函数私有化
内存保护修改是通用需求。函数会在未来 Patch 模块复用，移至 `utils.hpp` / `utils.cpp`。

## 中等问题

### 5. `mod_aim.cpp` — 魔法数字 `7`
```cpp

```
**建议**: `7` 代表 `PROT_READ | PROT_WRITE | PROT_EXEC`。值固定，但宏定义增加可读性。

### 6. `CMakeLists.txt` - `file(GLOB)` 更新滞后性
```cmake

```
**建议**: `GLOB` 只在 CMake 配置阶段执行。新增模块后需手动重新触发配置（修改 CMakeLists）。添加注释提醒。
