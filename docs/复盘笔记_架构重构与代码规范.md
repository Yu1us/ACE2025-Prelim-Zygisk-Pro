## 🔴 严重问题 (High Risk)

### 1. `registry.hpp` — 内存所有权模型有隐患
> [!CAUTION]
> `createAll()` 的返回结果存在误导。

```cpp
// L22-28
std::vector<IHackModule *> createAll() {
    std::vector<IHackModule *> modules;
    for (auto &f : factories_) {
        modules.push_back(f());  // f() 返回 static 局部变量的指针
    }
    return modules;
}
```
**分析**：`AutoRegister` 的 lambda 中使用了 `static T instance`，返回的是静态局部变量的指针。虽然不会内存泄漏，但 `createAll()` 这个名字暗示每次调用都会创建新实例。如果多次调用，会得到同一套指针，可能导致状态混乱。

**建议**：既然实际是单例模式，将方法名改为 `getAll()` 或 `modules()`，以匹配其实际行为。
