**日期**: 2026-01-18

## 1. Analysis Path (我的思路)

### 1.1 现象分析
- **现象**: 角色移动速度极快，且有奇怪的加速度。
- **推测**: UE4 的 `CharacterMovementComponent` 组件中，`MaxWalkSpeed` 被恶意修改。

### 1.2 寻找证据 (Writeup Reference)
- **工具**: UE4Dumper
- **关键结构**:
    - `ACharacter` -> `CharacterMovement` (Offset: 由 SDK 确认)
    - `UCharacterMovementComponent` -> `MaxWalkSpeed` (Offset: 由 SDK 确认)
- **NotebookLM 检索结果**:
    > "The offset was found by dumping the SDK... CharacterMovementComponent is at `0x288`, MaxWalkSpeed is at `0x18c`."

### 1.3 Offset Map

## 2. Logic & Prototype (Frida)

### 2.1 核心逻辑 (`scripts/solve_speed.js`)
我们使用 Frida 快速验证了修复逻辑：
1. 找到 `FirstPersonCharacter`。
2. 读取 `0x288` 指针。
3. 检查 `0x18c` 是否大于 600。
4. 强制写入 600。

```javascript
// 关键代码片段
const movementComp = character.add(0x288).readPointer();
const oldSpeed = movementComp.add(0x18c).readFloat();
if (oldSpeed > 600) {
    movementComp.add(0x18c).writeFloat(600.0);
}
```

## 3. Engineering Implementation (C++ Zygisk)

### 3.1 挑战点 (Interview Prep)
- **Q**: 为什么在 C++ 里不能直接 `ptr->speed = 600`？
- **A**: 指针链过长 (`Actor -> Component -> Speed`)，每一级都可能为空。在游戏循环中不做检查直接访问会立即导致 SIGSEGV 崩溃。Frida 会捕获异常，C++ 不会。
- **Solution**: 使用 `safeRead` 或 `mincore` 检查页面有效性。
