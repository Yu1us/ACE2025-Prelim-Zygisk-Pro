# 复盘笔记：移动速度异常 (Speed Hack)

**日期**: 2026-01-18
**标签**: `UE4`, `Dump SDK`, `Memory Patch`

## 1. 分析

### 1.1 现象
角色移动速度极快，加速度异常。UE4 的 `CharacterMovementComponent` 中 `MaxWalkSpeed` 被修改。

### 1.2 证据
**工具**: UE4Dumper
**关键结构**:
- `ACharacter` -> `CharacterMovement` (Offset: SDK 确认)
- `UCharacterMovementComponent` -> `MaxWalkSpeed` (Offset: SDK 确认)

Writeup: "CharacterMovementComponent at `0x288`, MaxWalkSpeed at `0x18c`."

### 1.3 Offset Map
| Class | Member | Offset | Type |
| :--- | :--- | :--- | :--- |
| `ACharacter` | `CharacterMovement` | `0x288` | `UCharacterMovementComponent*` |
| `UCharacterMovementComponent` | `MaxWalkSpeed` | `0x18c` | `float` |
| `UCharacterMovementComponent` | `MaxAcceleration`| `0x1a0` | `float` |

## 2. 原型验证 (Frida)

### 2.1 核心逻辑 (`scripts/solve_speed.js`)
Frida 验证修复逻辑：
1. 找到 `FirstPersonCharacter`
2. 读取 `0x288` 指针
3. 检查 `0x18c` 是否大于 600
4. 强制写入 600

```javascript
// 关键代码片段
const movementComp = character.add(0x288).readPointer();
const oldSpeed = movementComp.add(0x18c).readFloat();
if (oldSpeed > 600) {
    movementComp.add(0x18c).writeFloat(600.0);
}
```

---

## 3. C++ 实现 (Zygisk)

### 3.1 挑战
**Q**: 为什么 C++ 不能直接 `ptr->speed = 600`？
**A**: 指针链长 (`Actor -> Component -> Speed`)，每级可能为空。游戏循环中不检查直接访问会 SIGSEGV。Frida 捕获异常，C++ 不会。
**解决**: `safeRead` 或 `mincore` 检查页面有效性。
